# Sicherheitsrichtlinie

## Sicherheitslücken melden

Bitte melde Sicherheitslücken **nicht** über öffentliche GitHub Issues, da dies anderen Angreifern Informationen geben könnte.

Schreibe stattdessen eine E-Mail an die Projektmaintainer (Kontakt im GitHub-Profil).

Bitte enthülle Sicherheitslücken erst öffentlich, nachdem ein Fix veröffentlicht wurde (Responsible Disclosure).

## Was wir als Sicherheitslücke betrachten

- Unautorisierter Zugriff auf Daten anderer Teams
- Möglichkeit, fremde Trainingsdaten zu lesen oder zu manipulieren
- Schwachstellen in der Firebase-Authentifizierung

## Was wir nicht als Sicherheitslücke betrachten

- Bugs ohne Sicherheitsauswirkungen
- Fehlende Features
